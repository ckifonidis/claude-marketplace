# Migrate agent-step 1.0.0 → 1.1.0

> Severity: **additive**. Applied by `/pull-library`; readable standalone.

## What changed

Read pagination became a first-class library feature. A list read opts in with `pageable` on its
`ActionDef`; the runner then injects optional `page`/`pageSize` params, runs/serves the read, and
spreads a uniform envelope into the StepResult:

```ts
{ page, pageSize, totalCount, totalPages, hasMore, items, fromCache }
```

Two modes:

```ts
// SELF — executor returns the FULL set; runner slices + caches it in `pagedRead`.
get_things: { description: "…", paramsSchema: z.object({ q: z.string().optional() }), prereqs: [...], pageable: true }

// DELEGATE — backend pages; executor reads page/pageSize, returns the page + totalCount.
search_things: { …, paramsSchema: z.object({ q: z.string() }), pageable: "delegate" }
```

New public surface (all additive — nothing existing changed):
- New library file `paginate.ts`, re-exported from `index.ts`: `DEFAULT_PAGE_SIZE`, `MAX_PAGE_SIZE`,
  `clampPageSize`, `querySignature`, `pageRows`, `buildPageEnvelope`, and types `PageEnvelope`,
  `PagedCache`, `PageableSpec`.
- `ActionDef.pageable?: PageableSpec`.
- Library-managed `pagedRead` state slot (the self-mode reslice-cache).
- New construction check: a `pageable` action's `paramsSchema` must be a `z.object`.

`buildAgentStepTool` / `runSteps` / `Executor` / `Selector` signatures are **unchanged**.

## Why

Paginating a large list read was previously hand-rolled per tool (manual slicing + a bespoke cache
slot + a `querySignature` helper). Folding it into the runner makes every paginated read present the
same envelope, share one cache mechanism, and stay out of executor code — the executor just returns
the full set (self) or one page (delegate).

## Library replacement

The vendored `src/agent-step/*` is replaced wholesale by `/pull-library` (verbatim copy from the
toolkit), **including the new `paginate.ts` + `paginate.test.ts`**. No per-file action is required to
keep existing tools working — `pageable` is opt-in and nothing else changed.

## Manual follow-ups

- **Adopting `pageable` for an existing hand-rolled paginated read is optional** and a judgement call.
  If a tool already paginates by hand (manual page slicing + a custom cache slot), it keeps working
  untouched. To migrate it: make the executor return the full set in `resultBody.items` (self) or the
  page + `totalCount` (delegate), set `pageable` on the action, ensure its `paramsSchema` is a
  `z.object`, and drop the hand-rolled cache slot/helper in favour of the library `pagedRead`. Verify
  the model still re-pages via `page: n+1`.
