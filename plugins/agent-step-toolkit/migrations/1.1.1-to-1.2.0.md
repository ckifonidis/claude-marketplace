# Migrate agent-step 1.1.1 → 1.2.0

> Severity: additive. Applied by `/pull-library`; readable standalone.

## What changed

One export: `PagedCacheSchema` is now re-exported from `index.ts`. It was always exported from
`state.ts` (since 1.1.0), but the barrel omitted it while exporting its siblings
(`AwaitingInputSchema`, `CurrentFlowSchema`) — so consumers importing from
`./agent-step/index.js` couldn't reach it. No signature, behaviour, or runtime change.

## Why

The library-managed slot schemas are part of the public contract: hosts spread
`agentStepStateSpec` / `agentStepZodShape`, and occasionally need one slot's schema directly. With
the barrel incomplete, a host needing the paged-read schema had to hand-roll a structural copy —
exactly the drift the exported fragments exist to prevent. Surfaced by a contract audit ahead of
wider use.

## Library replacement

The vendored `src/agent-step/*` is replaced wholesale by `/pull-library` (verbatim copy from the
toolkit). The new re-export arrives with it.

**No `<transforms>`** — this is an additive release. Consumer tools need no changes; the library
replacement alone completes the upgrade.

## Manual follow-ups

Optional cleanup (recommended, not required): if the project's `src/state.ts` hand-declares the
library-managed slots — its own `AwaitingInputSchema` / `CurrentFlowSchema` / `PagedCacheSchema`
`z.object`s, or per-slot `Annotation` entries for `awaitingInput` / `currentFlow` / `pagedRead` —
replace them with the library fragments, which is what the bootstrap template now scaffolds:

```ts
import { agentStepStateSpec, agentStepZodShape } from "./agent-step/index.js";

export const AgentState = Annotation.Root({
  ...MessagesAnnotation.spec,
  ...agentStepStateSpec,        // instead of hand-declared awaitingInput/currentFlow/pagedRead
  // … per-tool slots …
});

export const AgentStateSchema: z.ZodObject<any> = MessagesZodState.extend({
  ...agentStepZodShape,         // instead of hand-rolled slot schemas
  // … per-tool slot schemas …
});
```

Hand-rolled copies keep compiling (the shapes are structurally identical today), but they can drift
from the runner's contract on future upgrades — the fragments cannot.
