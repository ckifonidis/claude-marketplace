# Migrate agent-step 1.3.0 → 1.3.1

> Severity: internal. Applied by `/pull-library`; readable standalone.

## What changed

`HANDBACK_SIGNALS` (and therefore the `handoff_type` + `handoff_metadata.service_type` that
`createHandoffNode` emits in terminate mode / the delegate-failure fallback) now uses the
middleware's canonical lowercase vocabulary: `off_topic` → `"off_topic"` (was `"OFF_TOPIC"`).

## Why

Cross-checked against the middleware's handoff-processor source: its type matching is
case-insensitive, so both spellings route identically — but the canonical strings are
lowercase, and emitting them exactly leaves nothing to chance with any future consumer.

## Library replacement

The vendored `src/agent-step/*` is replaced wholesale by `/pull-library` (verbatim copy from
the toolkit). No consumer transforms — the library replacement is the whole upgrade.

<transforms>
1. id: no-uppercase-signal-assertions
   target: consumer code or tests that compare against the emitted handoff signal (rare —
   the signal is produced and consumed inside the library/middleware)
   change: if any consumer test or client asserts `handoff_type === "OFF_TOPIC"` (or
   `service_type === "OFF_TOPIC"`), relax it to the lowercase canonical `"off_topic"` or a
   case-insensitive comparison.
   check: `grep -rn '"OFF_TOPIC"' src/` (outside `src/agent-step/`) returns nothing.
</transforms>
