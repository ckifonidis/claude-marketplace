# Migrate agent-step 1.1.0 → 1.1.1

> Severity: internal. Applied by `/pull-library`; readable standalone.

## What changed

Documentation only — doc-comments in the library source. No public-API, signature, export, or
runtime-behavior change. Specifically:

- `runner.ts` header: `cancel_pending_confirmation` → `abort_pending_input` (the header named a
  reserved action that never existed; the real one is `abort_pending_input`).
- `types.ts` `ConfirmationOpts.ttlMs`: now documented as **INERT** — accepted for forward-compat but
  never read; the runner times nothing out. The `CONFIRMATION_DEFAULTS.ttlMs` value in `runner.ts`
  is annotated as unused.
- `types.ts` `ExecutorResult.ok`: now documented as a **batch-continuation control flag, not a
  success/verdict signal** — returning `ok: true` for a "negative" domain outcome (with the verdict
  in `resultBody`) is valid when later steps should still run.
- `types.ts` `startsFlow`: now documents that a flow persists across turns and is cleared only by
  `endsFlow`/`abortFlow` — there is no implicit reset on a goal-switch; the host drives it.

The behaviour these comments describe was already true in 1.1.0; only the prose was wrong or
incomplete.

## Why

In preparation for wider use, the library's documented contract was audited against live runtime
behavior. The runtime conformed in every exercised path; the audit found only places where the
prose was misleading (a stale action name, `ttlMs` implied live, `ok` semantics under-specified) or
silent (flow persistence across a goal-switch). These corrections make the contract honest for
readers who did not write the library.

## Library replacement

The vendored `src/agent-step/*` is replaced wholesale by `/pull-library` (verbatim copy from the
toolkit). The refreshed doc-comments are the entire upgrade.

**No `<transforms>`** — this is an internal/doc-only release. Consumer tools need no changes; the
library replacement alone completes the upgrade.
