# Migrate agent-step 1.3.1 → 1.4.0

> Severity: additive. Applied by `/pull-library`; readable standalone.

## What changed

The built-in `request_handoff` (library handoff opt-in) accepts the full handback signal set:

- `HandoffRequest.reason`: `"off_topic" | "completed" | "abandon"` (was `"off_topic"` only).
- `context` is per-reason — the customer's request for `off_topic`; the **LLM-composed
  closing line** for `completed` / `abandon`, delivered verbatim by the resolver node as the
  final reply with the reason's signal in `handoff_type`. `completed` / `abandon` never
  delegate. `context` now requires non-empty (`min(1)`).
- `HANDBACK_SIGNALS` extends to all three (identity mapping, canonical lowercase).

## Why

A specialized agent's role policy needs all three handbacks (see
`streaming-and-channel-contract.md` `<agent_roles>`): `completed` after the wrap-up,
`abandon` when the user bails, `off_topic` on a substantive topic change. The closing for
`completed` / `abandon` is LLM-composed so it can reference what was actually done — matching
the middleware's passive handback semantics (reply delivered; routing flips next request).

## Library replacement

The vendored `src/agent-step/*` is replaced wholesale by `/pull-library` (verbatim copy from
the toolkit). No consumer transforms — the library replacement is the upgrade. The test suite
